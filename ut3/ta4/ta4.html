Exercise 1

  1. Import the training dataset ("cardiac-training.csv") into RM.
    • Set the attribute "2do_Ataque_Corazon" according to the problem. Why do we do this?

      We set the attribute to have values 1 and 0 for "Yes" and "No" respectively, which is necessary for the logistic regression model that requires all its inputs and outputs to be numeric variables.

  2. Import the test/evaluation/prediction file ("cardiac-scoring.csv").
    • Verify that the data type of the attributes is "integer". Why do we do this?

      Ensuring that all attributes are of type "integer" allows us to use the linear regression model to interpret all independent variables and predict a specific input data point. If the data were not numeric, we could not use linear regression.

  3. Run the model and compare the attribute ranges between the training and evaluation datasets.
    • How are these ranges comparably?

      The ranges are the same.

    • Are all the attributes of the evaluation/prediction examples within the attribute ranges of the training dataset?

      Yes.

    • Why do we need to verify this?

      This allows us to ensure that the data is consistent throughout the training and testing process of the model. This way, we make sure there are no noise or biases that interfere with the predictions obtained.

    • Are there more data preparation tasks to be done?

      We should check the data distribution, normalize it, and generally clean up the presence of noise, outliers, correlated variables, replace or eliminate missing values, etc.
